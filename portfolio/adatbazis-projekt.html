Automatizált Időjárási Adatok Feldolgozása

Zátrok Márton

1. A projekt célja és feladata

A projektfeladat célja az volt, hogy a "Mikrovezérlők programozása" tantárgy keretében elkészített, saját építésű adatgyűjtő eszköz által mért adatokat összevessem egy nyilvánosan elérhető, professzionális légszennyezettség-mérő állomás adataival. Az összehasonlításhoz a Kisgyőr településen található, "Sz-R.cs" azonosítójú openSenseMap szenzor adatait használtam.

Fő célkitűzések:

  • Adatok gyűjtése két különböző forrásból (saját és külső).
  • Az összegyűjtött adatok feldolgozása, tisztítása és egységes formátumra hozása.
  • Egy relációs adatbázis megtervezése és létrehozása az adatok strukturált tárolására.
  • Az adatok betöltése az adatbázisba.
  • SQL lekérdezések segítségével az adatok elemzése, statisztikai mutatók (minimum, maximum, átlag) számítása és a két adatsor összevetése.
  • Az eredmények kiértékelése és a teljes folyamat dokumentálása.

2. Technikai megvalósítás

Adatforrások

1. Saját mérési adatok: A saját adataimat egy általam épített és programozott mikrovezérlő alapú készülékkel gyűjtöttem (RP2040).

  • DHT11 szenzor: Hőmérséklet és relatív páratartalom mérő
  • LDR: Fényerő mérő szenzor.
  • Feszültségosztó áramkör: A rendszer tápfeszültségének monitorozása

2. Külső adatok (openSenseMap): A külső referenciaadatok forrása az openSenseMap online adatplatform volt. Az Sz-R.cs azonosítójú kisgyőri szenzorboxból töltöttem le CSV formátumban az alábbi adatokat:

  • Hőmérséklet (°C)
  • Relatív páratartalom (%)
  • Légnyomás (Pa)
  • PM4.0 szálló por koncentráció (μg/m³)
  • Tápfeszültség (V)

Minden adattípus külön CSV fájlban volt elérhető, createdAt (időbélyeg) és value (mért érték) oszlopokkal.

Adatbázis Logikai Terv

Az adatbázis tervezésének alapelve a két eltérő forrásból származó adathalmaz logikai szétválasztása, de egyúttal a hatékony összehasonlíthatóság biztosítása volt.

A modell két fő entitásból (táblából) épül fel, melyeket a Dátum kapcsol össze:

  • saját_mérési_adatok: Datum, Homerseklet, Paratartalom, Fenyero, Feszultseg
  • kulso_forras: Datum, Legnyomas, Homerseklet, Feszultség, Por_Pollen, Paratartalom

Fizikai megvalósítás és adatfeltöltés

Az adatbázis fizikai létrehozása XAMPP szerverkörnyezetben, a phpMyAdmin adminisztrációs felületén történt. Az adatimportálás legösszetettebb lépése a külső forrásból származó, öt különálló CSV fájl egységes táblába rendezése volt. Ezt a Thonny alkalmazásban megírt Python programokkal oldottam meg.

➤ Python: csv file time adat szabályzó.py

A megadott Python program egy CSV fájlban tárolt időbélyeggel ellátott adatokat dolgoz fel, és egy új, megtisztított és kiegészített CSV fájlt hoz létre. Célja: Időbélyegek normalizálása, hiányzó percek pótlása.

import csv from datetime import datetime, timedelta import os from tkinter import Tk from tkinter.filedialog import askopenfilename def parse_timestamp(ts_str): """ Időbélyeg string feldolgozása datetime objektummá. Támogatja az ISO 8601 formátumot (pl. '2024-03-10T21:15:34.078Z'). """ try: if ts_str.endswith('Z'): return datetime.fromisoformat(ts_str[:-1] + '+00:00') else: return datetime.fromisoformat(ts_str) except ValueError: try: return datetime.strptime(ts_str, "%Y-%m-%d %H:%M:%S.%f") except ValueError: try: return datetime.strptime(ts_str, "%Y-%m-%dT%H:%M") # Ha csak óra:perc van T-vel except ValueError: print(f"Figyelmeztetés: Nem sikerült feldolgozni az időbélyeget: {ts_str}") return None def format_timestamp(dt_obj): """ Datetime objektum formázása ISO 8601 stringgé, másodpercek nélkül (00), UTC jelöléssel. """ return dt_obj.strftime("%Y-%m-%dT%H:%M:00Z") def process_csv(input_filepath, output_filepath): """ Feldolgozza a bemeneti CSV fájlt és létrehozza a kimeneti CSV-t a megadott logika szerint. Az értékeket két tizedesjegyre kerekíti. """ data = [] try: with open(input_filepath, mode='r', newline='', encoding='utf-8') as infile: reader = csv.DictReader(infile) if 'createdAt' not in reader.fieldnames or 'value' not in reader.fieldnames: print("Hiba: A CSV fájlnak tartalmaznia kell 'createdAt' és 'value' oszlopokat.") return False for row_number, row in enumerate(reader, 1): timestamp_str = row.get('createdAt') value_str = row.get('value') if timestamp_str is None or value_str is None: print(f"Figyelmeztetés (sor {row_number}): Sor kihagyva hiányzó 'createdAt' vagy 'value' miatt: {row}") continue dt_obj = parse_timestamp(timestamp_str) if dt_obj: try: value = float(value_str) dt_obj_minute = dt_obj.replace(second=0, microsecond=0) data.append({'timestamp': dt_obj_minute, 'value': value}) # Eredeti érték mentése itt except ValueError: print(f"Figyelmeztetés (sor {row_number}): Nem sikerült a '{value_str}' értéket számmá alakítani. 1. Lekérdezés: Min, Max és Átlag hőmérsékletek

Min hőmérséklet és Max hőmérséklet valamint Atlag hőmérséklet külön nézve mind a kettő táblán. Az összes érték 2 tizedesjegyre van megadva.

SELECT 'Saját mérés' AS Forras, MIN(Homerseklet) AS Min_Ho, MAX(Homerseklet) AS Max_Ho, ROUND(AVG(Homerseklet), 2) AS Atlag_Ho FROM saját_mérési_adatok UNION ALL SELECT 'Kulso forras' AS Forras, MIN(Homerseklet), MAX(Homerseklet), ROUND(AVG(Homerseklet), 2) FROM kulso_forras;

2. Lekérdezés: Szélsőértékek pontos dátummal

A külső és a saját adatokat összevetve írja ki a maximum és minimum hőmérséklet és páratartalom értéket és írja hozzá a pontos dátumot, hogy mikor volt ez a mérés.

( SELECT 'Minimum' AS Tipus, Datum, Homerseklet, Paratartalom FROM ( SELECT Datum, Homerseklet, Paratartalom FROM ( SELECT Datum, Homerseklet, Paratartalom FROM kulso_forras UNION ALL SELECT Datum, Homerseklet, Paratartalom FROM saját_mérési_adatok ) AS osszes ORDER BY Homerseklet ASC, Paratartalom ASC, Datum ASC LIMIT 1 ) AS minadat ) UNION ( SELECT 'Maximum' AS Tipus, Datum, Homerseklet, Paratartalom FROM ( SELECT Datum, Homerseklet, Paratartalom FROM ( SELECT Datum, Homerseklet, Paratartalom FROM kulso_forras UNION ALL SELECT Datum, Homerseklet, Paratartalom FROM saját_mérési_adatok ) AS osszes ORDER BY Homerseklet DESC, Paratartalom DESC, Datum ASC LIMIT 1 ) AS maxadat );

3. Lekérdezés: Minimum és maximum Por/Pollen érték

A külső forrásból a Minimum és maximum Por_Pollen érték IDŐPONTTAL együtt (12:00–16:00 között).

( SELECT 'Minimum' AS Tipus, Datum, Por_Pollen FROM kulso_forras WHERE TIME(Datum) BETWEEN '12:00:00' AND '16:00:00' ORDER BY Por_Pollen ASC LIMIT 1 ) UNION ( SELECT 'Maximum' AS Tipus, Datum, Por_Pollen FROM kulso_forras WHERE TIME(Datum) BETWEEN '12:00:00' AND '16:00:00' ORDER BY Por_Pollen DESC LIMIT 1 );

4. Lekérdezés: Időszaki átlagok (20 perces bontás)

A kulso_forras és a saját_mérési_adatok táblákból kellett lekérdezni a 2025. március 19-én 12:00 és 16:00 közötti időszakra vonatkozóan a hőmérséklet és a páratartalom értékek átlagát, 20 perces bontásban.

SELECT DATE_FORMAT( CONCAT( DATE(Datum), ' ', LPAD(HOUR(Datum), 2, '0'), ':', LPAD(FLOOR(MINUTE(Datum) / 20) * 20, 2, '0'), ':00' ), '%Y-%m-%d %H:%i:%s' ) AS Idopont, ROUND(AVG(Homerseklet), 2) AS Atlag_Homerseklet, ROUND(AVG(Paratartalom), 2) AS Atlag_Paratartalom FROM ( SELECT Datum, Homerseklet, Paratartalom FROM kulso_forras WHERE TIME(Datum) BETWEEN '12:00:00' AND '16:00:00' UNION ALL SELECT Datum, Homerseklet, Paratartalom FROM saját_mérési_adatok WHERE TIME(Datum) BETWEEN '12:00:00' AND '16:00:00' ) AS egyesitett GROUP BY Idopont ORDER BY Idopont;
